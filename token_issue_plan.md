# 任务计划：深度排查 token 失效造成的权限不足问题

## 目标
系统性排查并解决 token 失效时被误报为"权限不足"的问题，确保用户看到准确的错误提示。

## 背景
- **问题描述**：当 token 过期或失效时，系统返回"权限不足"（403），而不是"登录已过期"（401）
- **用户体验影响**：用户不知道需要重新登录，误以为是权限配置问题
- **影响范围**：所有使用 `checkPermission` 中间件的 API 路由

## 当前状态
已完成的工作：
- ✅ 添加了 `AuthStatus.EXPIRED_TOKEN` 状态
- ✅ 在 `authenticateUser` 中区分过期和无效
- ✅ 在 `checkPermission` 中返回不同的错误消息
- ✅ 在 `lib/fetch.ts` 中添加 401 拦截器
- ✅ 在 `useCurrentUser` 中处理 401 错误

**待排查**：为什么用户仍然反馈看到"权限不足"
