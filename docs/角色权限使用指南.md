# 小牛好学管理系统 — 角色权限使用指南

## 系统概览

系统采用基于角色的访问控制（RBAC），共定义 **9 个角色**、**12 类资源**、**20 种操作**。

权限通过 `lib/permissions.ts` 中的权限矩阵统一管理，前端通过侧边栏和路由守卫控制页面访问，后端通过 API 中间件校验操作权限。

---

## 角色总览

| 角色 | 标识 | 定位 |
|------|------|------|
| 超级管理员 | `admin` | 系统全局管理，不参与日常业务 |
| 运营人员 | `operator` | 线索录入与分配、待办管理 |
| 销售顾问 | `sales` | 线索跟进、试听转化、正式订单录入 |
| 班主任 | `head_teacher` | 学生管理、排课、回访、续费 |
| 教师 | `teacher` | 查看课节、录入教学信息 |
| 教务 | `academic_affairs` | 教学管理、排课、老师入库、课时核对 |
| 财务 | `finance` | 财务管理、老师打款、入库确认 |
| 招师 | `teacher_recruiter` | 候选人约面、初试录入、录像上传 |
| 人事 | `hr` | 入库确认、业绩核对 |

## 各角色权限详解

### 1. 超级管理员（admin）

**定位**：系统全局管理，保留管理和查看权限，不参与日常业务操作。

**侧边栏可见模块**：
- 控制台首页
- 客户管理（线索跟进、公共线索池）
- 订单管理（试听课、正式课、老师库销售版）
- 招师管理（面试管理、老师约面、初试录像上传、储备候选人）
- 教务管理（全部子菜单）
- 质检系统（试听转化质检、课后服务质检）
- 待办事项
- 系统管理（字典、用户管理、角色管理）

**权限矩阵**：

| 资源 | 可执行操作 |
|------|-----------|
| 线索 | 查看、创建、编辑、删除 |
| 试听课 | 查看、编辑、删除、添加上课链接 |
| 学生 | 查看、创建、编辑、删除 |
| 正式订单 | 查看、创建、编辑、删除 |
| 课节 | 查看、创建、编辑、删除 |
| 课程 | 查看、创建、编辑、删除 |
| 课程异动 | 查看、创建、编辑、核对课时、打款、核对业绩 |
| 老师面试 | 查看、创建、约面、评价、入库确认、录像上传、录像复核、删除 |
| 老师库 | 查看、创建、编辑、删除、备注 |
| 字典 | 查看、创建、编辑、删除 |
| 用户 | 查看、创建、编辑、删除 |
| 待办 | 查看、创建、编辑、删除 |
| 上传 | 创建 |

**使用指南**：
- 管理系统用户账号、角色分配
- 管理字典数据（地区、年级、科目等）
- 处理异常数据和需要删除的记录
- 不直接参与线索反馈、试听创建等日常业务流程

## 角色协作流程

### 流程一：线索→试听→正式订单（销售主导）

```
运营人员                销售顾问               教务                  班主任
    │                      │                    │                     │
    ├─ 录入线索 ──────────→│                    │                     │
    │                      ├─ 跟进反馈          │                     │
    │                      ├─ 创建试听课 ──────→│                     │
    │                      │                    ├─ 匹配老师            │
    │                      │                    ├─ 确认老师(ClassIn)   │
    │                      ├─ 确认试听时间      │                     │
    │                      ├─ 试听成功          │                     │
    │                      ├─ 创建正式订单 ────→│                     │
    │                      │                    │                     ├─ 排课
    │                      │                    │                     ├─ 回访
```

**关键交接点**：
- 运营→销售：线索分配（通过待办或公共线索池抢单）
- 销售→教务：试听课创建后，教务在「待试听匹配」中匹配老师
- 销售→班主任：正式订单创建后，班主任接手排课和学生服务

### 流程二：老师招聘→入库（招师主导）

```
招师                    教务                   财务                  人事
  │                       │                     │                     │
  ├─ 创建候选人            │                     │                     │
  ├─ 约面                  │                     │                     │
  ├─ 初试评价              │                     │                     │
  ├─ 上传录像 ────────────→│                     │                     │
  │                       ├─ 录像复核            │                     │
  │                       ├─ 教学评价            │                     │
  │                       │                     │                     │
  │                       ├─ 入库确认 ──────────→├─ 入库确认 ──────────→├─ 入库确认
  │                       │                     │                     │
  │                       ├─ 老师入库到老师库    │                     │
```

**关键交接点**：
- 招师→教务：录像上传后，教务复核教学能力
- 教务→财务→人事：三方依次确认入库（财务确认打款信息，人事确认合同信息）
- 入库完成后老师进入「老师库」，销售和教务可为其匹配试听

### 流程三：排课与课时管理（班主任+教务）

```
班主任                  教务                   财务
  │                       │                     │
  ├─ 为学生排课            │                     │
  ├─ 创建课节              │                     │
  ├─ 编辑课程              │                     │
  │                       ├─ 核对课时            │
  │                       ├─ 课程异动处理        │
  │                       │                     ├─ 老师打款
  │                       ├─ 核对业绩            │
```

**关键交接点**：
- 班主任排课后，教务核对课时数据
- 教务确认课时后，财务执行打款

## 技术实现说明

### 权限配置位置

- **权限矩阵**：`lib/permissions.ts` → `PERMISSION_MATRIX`
- **侧边栏菜单**：`components/dashboard/sidebar.tsx` → `navigationGroups`
- **路由守卫**：`lib/dashboard-route-access.ts` → `canAccessDashboardRoute`
- **API 中间件**：`proxy.ts`（原 middleware）→ 校验 `hasPermission`
- **前端 Hook**：`lib/hooks/usePermission.ts` → `usePermission()`

### 权限检查方式

```typescript
// 前端组件中
import { usePermission } from '@/lib/hooks/usePermission'

const { trialLessons } = usePermission()
if (trialLessons.matchTeacher()) {
  // 显示匹配老师按钮
}

// 后端 API 中
import { hasPermission } from '@/lib/permissions'

if (!hasPermission(profile.role, 'trialLessons', 'matchTeacher')) {
  return NextResponse.json({ error: '权限不足' }, { status: 403 })
}
```

### 新增角色或权限时的步骤

1. 在 `lib/permissions.ts` 的 `ROLES` 中添加角色标识
2. 在 `PERMISSION_MATRIX` 中添加该角色的权限配置
3. 在 `components/dashboard/sidebar.tsx` 的 `navigationGroups` 中将该角色加入相应菜单组的 `roles` 数组
4. 在 `lib/dashboard-route-access.ts` 中配置该角色可访问的路由规则
5. 在 `proxy.ts` 中确保 API 权限校验覆盖新角色
