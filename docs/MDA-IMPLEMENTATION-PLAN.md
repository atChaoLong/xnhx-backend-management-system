## 2. MDA架构设计

### 2.1 核心理念

**模型驱动架构(Model-Driven Architecture)**:
1. **数据模型优先** - 所有功能基于数据模型定义
2. **状态机驱动** - 业务状态由状态机统一管理
3. **权限模型化** - 基于RBAC的声明式权限控制
4. **自动化计算** - 状态和统计通过触发器和函数自动计算

### 2.2 架构分层

```
┌─────────────────────────────────────────────┐
│           Presentation Layer                │
│  (Next.js Pages + shadcn/ui Components)     │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│          Business Logic Layer               │
│  (Status Calculators + Services + Hooks)    │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│            Data Access Layer                │
│     (Supabase Client + RLS Policies)        │
└─────────────────┬───────────────────────────┘
                  │
┌─────────────────▼───────────────────────────┐
│              Data Model Layer                │
│   (PostgreSQL Tables + Views + Triggers)    │
└─────────────────────────────────────────────┘
```

## 4. 技术债务处理

### 4.1 代码重构

#### 4.1.1 服务层统一
```typescript
// 统一服务层接口
interface BaseService<T> {
  getAll(filters?: object): Promise<T[]>;
  getById(id: string): Promise<T>;
  create(data: NewType<T>): Promise<T>;
  update(id: string, data: Partial<NewType<T>>): Promise<T>;
  delete(id: string): Promise<void>;
}
```

#### 4.1.2 状态管理优化
- 使用React Query管理服务端状态
- 使用Zustand管理全局状态
- 统一错误处理

### 4.2 数据库优化

#### 4.2.1 索引优化
```sql
-- 线索表索引
CREATE INDEX idx_leads_add_status ON leads(add_status);
CREATE INDEX idx_leads_convert_status ON leads(convert_status);
CREATE INDEX idx_leads_operator ON leads(operator_id);

-- 订单表索引
CREATE INDEX idx_orders_student ON orders(student_id);
CREATE INDEX idx_orders_type ON orders(order_type);

-- 学生表索引
CREATE INDEX idx_students_status ON students(status);
```

### 4.3 安全加固

#### 4.3.1 RLS策略完善
```sql
-- 线索表RLS
CREATE POLICY "Operators can manage their leads" ON leads
  FOR ALL USING (auth.uid()::text = operator_id);
```
